                        SNIP — URL SHORTENER
                   Complete Project Documentation
================================================================================

PROJECT OVERVIEW
----------------
Snip is a full-stack URL shortener with real-time click analytics, Redis caching,
background job processing, rate limiting, and event-driven click tracking via Kafka.
Users paste a long URL, receive a short URL + QR code instantly, can optionally set
a custom alias, expiry date, password protection, max-clicks limit, and webhook URL.
Authenticated users get a personal dashboard and a programmatic API key.

Repository layout:
  /Backend    — NestJS REST API
  /Frontend   — Next.js web application

FEATURE SET
-----------
  1. URL Shortening             — nanoid(6) codes or custom aliases (4–10 chars)
  2. QR Code Generation         — Indigo-on-dark QR rendered to data URI at creation time
  3. Password-Protected URLs    — bcrypt-hashed password; unlock page gates redirect
  4. Max-Clicks (self-destruct) — link goes 410 Gone after N clicks
  5. Expiry Dates               — ISO 8601 TTL; smart cache respects expiry
  6. User Authentication        — JWT register/login + optional auth on URL creation
  7. API Key Authentication     — snip_* keys; stored as SHA-256 hash, shown once
  8. Google Safe Browsing       — Real-time malware/phishing detection on all links
  9. Webhook Integration        — POST click payload to user-defined URL on every click
 10. Kafka Click Pipeline       — Event-driven click tracking with Dead Letter Queue
 11. GeoIP Enrichment           — ip-api.com lookup: country + city per click
 12. UA Parsing                 — ua-parser-js: device type, browser, OS per click
 13. ClickEvent Records         — per-click DB rows with all enrichment fields
 14. Analytics Breakdowns       — top-10 browsers, devices, countries from ClickEvents
 15. Time-Series Analytics      — 30-day clicks-per-day bar chart with hover tooltips
 16. Redis Distributed Lock     — setNx prevents race conditions on alias creation
 17. Idempotency Support        — Idempotency-Key header deduplicates URL creation
 18. Bulk Shortening            — frontend page shortens up to 20 URLs at once
 19. Observability Stack        — Prometheus metrics + Grafana visualization

AUTH SECURITY FEATURES
----------------------
 20. HttpOnly Secure Cookie     — refresh token set as httpOnly; SameSite=Strict cookie;
                                  inaccessible to JavaScript / XSS attacks
 21. Refresh Token Rotation     — every refresh issues a new token pair; old token deleted
 22. Token Family Tracking      — each session has a familyId; replaying a rotated token
                                  revokes the entire family (replay / theft detection)
 23. Login Brute-Force Lock     — 5 failed attempts → 15-min lockout stored in Redis;
                                  counter cleared on successful login
 24. Access Token Revocation    — jti (UUID) embedded in JWT; added to Redis blocklist
                                  on logout; JwtStrategy rejects blocklisted tokens
                                  immediately (not just after natural expiry)
 25. API Key Hashing            — SHA-256 hash stored in DB; plaintext returned once;
                                  guard hashes incoming key before DB lookup

ENTRY POINT: src/main.ts
--------------------------
The bootstrap function:
  1. Creates the NestJS application
  2. Enables graceful shutdown hooks (SIGTERM drains in-flight requests)
  3. Registers cookie-parser middleware (for httpOnly refresh token)
  4. Configures CORS — origin from FRONTEND_URL, credentials: true
  5. Applies global ValidationPipe — strips unknown fields, rejects bad input
  6. Mounts Swagger UI at /api/docs
  7. Connects and starts the Kafka microservice consumer (click-events topic)
  8. Starts HTTP server on PORT env var (default 3001)
  9. Imports batch.processor.ts as a side-effect so the Redis→DB flush job
     runs in the same process (suitable for single-dyno deploys)

DATABASE SCHEMA (prisma/schema.prisma)
---------------------------------------
  model User {
    id          String    @id @default(cuid())
    email       String    @unique
    password    String                          — bcrypt(12)-hashed
    apiKey      String?   @unique               — SHA-256 hash of snip_* key
    createdAt   DateTime  @default(now())
    updatedAt   DateTime  @updatedAt
    urls        Url[]
  }

  model Url {
    id          String       @id @default(cuid())
    shortCode   String       @unique
    originalUrl String
    expiresAt   DateTime?                       — optional TTL
    clicks      Int          @default(0)        — flushed click counter
    maxClicks   Int?                            — self-destruct threshold
    password    String?                         — bcrypt-hashed, nullable
    qrCode      String?                         — data URI (generated at creation)
    webhookUrl  String?                         — POSTed on each click
    userId      String?                         — nullable FK to User
    user        User?        @relation(...)     — onDelete: SetNull
    clickEvents ClickEvent[]
    createdAt   DateTime     @default(now())
    updatedAt   DateTime     @updatedAt

    @@index([userId])
  }

  model ClickEvent {
    id        String   @id @default(cuid())
    urlId     String
    url       Url      @relation(...)           — onDelete: Cascade
    country   String?                           — from GeoService
    city      String?
    device    String?                           — from ua-parser-js (desktop/mobile/tablet)
    browser   String?
    os        String?
    createdAt DateTime @default(now())

    @@index([urlId])
  }

BACKEND — MODULES IN DETAIL

── AUTH MODULE ─────────────────────────────────────────────────────────────────

  AuthController
    POST /auth/register         — creates user; access_token in body,
                                  refresh_token as httpOnly cookie
    POST /auth/login            — validates credentials; same token split as register
    POST /auth/refresh          — reads refresh_token cookie; validates, rotates pair;
                                  returns new access_token in body + new cookie
    POST /auth/logout           — revokes refresh family + blocklists access token jti;
                                  clears cookie
    GET  /auth/api-key          — (JwtAuthGuard) returns { hasApiKey: bool }
    POST /auth/api-key/generate — (JwtAuthGuard) generates key; returns plaintext ONCE

  AuthService
    register(dto)           — checks email uniqueness, bcrypt(12) hash, creates User,
                              calls generateTokens()
    login(dto)              — checkBruteForce(), finds user, bcrypt.compare,
                              clearLoginAttempts(), calls generateTokens()
    generateTokens(userId, email, familyId?)
                            — signs JWT with jti (crypto.randomUUID()) + 15m TTL;
                              creates refresh token (32 random bytes hex), stores
                              JSON {userId, familyId} in Redis with 7-day TTL
    refresh(refreshToken)   — reads stored payload, checks revoked_family key,
                              deletes old token, calls generateTokens() with same familyId
    logout(refreshToken, accessToken?)
                            — sets revoked_family:{familyId} in Redis (7-day TTL);
                              decodes access token jti, writes to blocklist:{jti}
                              with TTL = remaining token lifetime
    checkBruteForce(email)  — reads login_locked:{email} from Redis; throws 429 if set
    recordFailedAttempt(email)
                            — redis.incr(login_attempts:{email}); sets 15-min expire
                              on first attempt; after 5 failures writes login_locked:{email}
    generateApiKey(userId)  — crypto.randomBytes(24).hex → snip_ prefix; SHA-256 hash
                              stored in DB; plaintext returned once
    getApiKey(userId)       — returns { hasApiKey: bool }; never exposes hash

  Guards
    JwtAuthGuard          — standard passport-jwt guard; throws 401 if missing/invalid
    OptionalJwtAuthGuard  — extends JwtAuthGuard; does NOT throw on missing token
    ApiKeyGuard           — reads x-api-key header; SHA-256 hashes it; prisma lookup by hash

  JWT Strategy (passport-jwt)
    Reads JWT_SECRET from env; extracts Bearer from Authorization header.
    validate(payload): checks blocklist:{jti} in Redis before returning user.
    Payload: { sub: userId, email, jti }

  Refresh Token Storage (Redis keys)
    refresh:{token}          → JSON { userId, familyId }  TTL 7 days
    revoked_family:{familyId}→ "1"                         TTL 7 days
    blocklist:{jti}          → "1"                         TTL = remaining access token life
    login_attempts:{email}   → count string                TTL 15 min (sliding window)
    login_locked:{email}     → "1"                         TTL 15 min

── URL MODULE ──────────────────────────────────────────────────────────────────

  UrlController   POST   /url              — create (optional auth via JWT or API key)
                  GET    /url/my-links     — (JwtAuthGuard) list URLs owned by user
                  DELETE /url/:code        — (JwtAuthGuard) delete owned URL
                  POST   /url/:code/unlock — verify password for protected URL
                  GET    /url/:code/stats  — delegates to AnalyticsService

  UrlService.createShortUrl(dto, idempotencyKey?)
    Inputs: originalUrl, customAlias?, expiresAt?, maxClicks?, password?,
            webhookUrl?, userId?

    Idempotency check (if Idempotency-Key header present):
      Reads Redis key "idemp:{key}". Returns cached JSON immediately on hit.

    1. Validates expiresAt is in the future (throws 400 if not)
    2. Calls checkSpam(url) — Google Safe Browsing API (skipped if no API key)
    3. Hashes password with bcrypt(10) if provided
    4. Loop up to MAX_RETRIES=5:
         a. Generates shortCode: customAlias or nanoid(6)
         b. Generates QR code data URI (qrcode.toDataURL, indigo+dark theme)
         c. Acquires Redis distributed lock: setNx("url-lock:{code}", "LOCK", 5s TTL)
            — If lock fails on custom alias → throws 400 immediately
            — If lock fails on nanoid → warns and retries
         d. prisma.url.create(...)
            — P2002 on custom alias → throws 400 "Alias already taken"
            — P2002 on nanoid → retries with new code
         e. On success: builds result { shortCode, shortUrl, qrCode, hasPassword, maxClicks }
         f. If idempotencyKey present: caches result in Redis for 24h
         g. Releases lock (finally block)
    5. After 5 failures → throws 500

── REDIRECT MODULE ─────────────────────────────────────────────────────────────

  RedirectController  GET /:code
    1. Checks RESERVED_PATHS set {'health','metrics','url','auth','api'} → 404
    2. Rate-limits by IP via RateLimiterService
    3. Calls RedirectService.getOriginalUrl(code)
    4. If result.requiresPassword → returns 200 JSON { requiresPassword: true, shortCode }
    5. Extracts real IP from x-forwarded-for header
    6. Publishes click event to Kafka (fire and forget)
    7. Returns HTTP 302 redirect to original URL

  RedirectService.getOriginalUrl(code) → RedirectResult
    1. Queries PostgreSQL for the URL record
    2. If not found → throws 404
    3. If expiresAt < now → throws 410 Gone
    4. If url.password is set → returns { requiresPassword: true }
    5. Max-clicks check: Redis counter + url.clicks >= url.maxClicks → throws 410
    6. Checks Redis cache → hit: returns immediately
    7. Calculates smart TTL: min(3600, secondsUntilExpiry)
    8. Caches url.originalUrl in Redis with smart TTL
    9. Returns { requiresPassword: false, originalUrl }

── ANALYTICS MODULE ────────────────────────────────────────────────────────────

  AnalyticsService.getStats(code)
    1. Queries PostgreSQL for URL + last 500 ClickEvents (desc)
    2. Throws 404 if URL not found
    3. Reads Redis counter "clicks:{code}" for pending clicks
    4. Aggregates ClickEvents into browserCounts, deviceCounts, countryCounts
    5. Converts to top-10 sorted lists: [{ label, count }]
    6. Queries ClickEvents for last 30 days (select createdAt only)
    7. Groups by day (YYYY-MM-DD); fills zeros for days with no clicks
    8. Returns:
         { shortCode, originalUrl, totalClicks, dbClicks, pendingClicks,
           createdAt, expiresAt, maxClicks, hasPassword, qrCode,
           browsers, devices, countries,
           clicksPerDay: [{ date: "YYYY-MM-DD", count: number }] × 30 }

── REDIS MODULE (SHARED CONNECTION) ────────────────────────────────────────────

  RedisService — thin wrapper around ioredis:
    get(key)                  — GET
    set(key, value, ttl=3600) — SET EX
    del(key)                  — DEL
    incr(key)                 — INCR
    setNx(key, value, ttlSec) — SET NX EX (returns boolean)
    expire(key, ttlSec)       — EXPIRE (used by brute-force counter)

  RateLimiterService
    10 requests per 60 seconds per IP. Throws 429 on breach.

── KAFKA MODULE ────────────────────────────────────────────────────────────────

  KafkaProducerService
    publishClickEvent(code, ip, userAgent): emits to "click-events" topic
    publishToDLQ(payload, errorMessage): emits to "click-events-dlq"

  KafkaConsumerService
    @EventPattern('click-events') handleClickEvent:
      1. redis.incr("clicks:{code}")
      2. UA parse → device, browser, OS
      3. GeoService.lookup(ip) → country, city
      4. prisma.clickEvent.create(all enrichment fields)
      5. triggerWebhook() if configured → DLQ on error

── QUEUE SYSTEM ────────────────────────────────────────────────────────────────

  Click tracking flow:
    GET /:code
    └─ kafkaProducer.publishClickEvent(code, ip, ua)
       └─ KafkaConsumerService.handleClickEvent()
          ├─ redis.incr("clicks:{code}")
          ├─ UA parse + GeoIP lookup
          ├─ prisma.clickEvent.create()
          └─ triggerWebhook() if configured
              [on error] → DLQ

  Batch Processor (runs every 10 seconds)
    SCAN Redis for "clicks:*" → prisma.url.updateMany({ clicks: { increment } })
    Deletes key ONLY after successful DB write.

── HEALTH & OBSERVABILITY ──────────────────────────────────────────────────────

  GET /health    → { status: 'ok', timestamp: ISO8601 }
  GET /api/docs  → Full Swagger UI (OpenAPI 3.0)
  GET /api/metrics → Prometheus metrics endpoint

BACKEND — OPTIMIZATIONS
================================================================================

1. REDIS CACHE FOR REDIRECTS
   Hot URLs serve from memory — no DB query. Latency <1ms vs. 5–20ms.

2. SMART CACHE TTL FOR EXPIRING URLS
   TTL is min(3600, secondsUntilExpiry). Never serves stale expired URLs.

3. REDIS SCAN INSTEAD OF KEYS (batch processor)
   redis.scan() with cursor pagination (COUNT 100) vs. blocking redis.keys() O(N).

4. WRITE BATCHING (batch processor)
   1000 clicks across 10 URLs = 10 DB writes per interval instead of 1000.

5. ASYNC CLICK TRACKING (Kafka)
   Redirect responds before click is counted. Kafka persists to disk.

6. RACE-CONDITION-FREE URL CREATION
   Redis distributed lock (setNx 5s TTL) + DB unique index backstop.

7. NANOID COLLISION RETRY (up to 5 attempts)
   64^6 ≈ 68.7 billion codes. Retry handles the vanishing edge case.

8. IDEMPOTENCY (24-hour Redis cache)
   Idempotency-Key header prevents duplicate URL creation on retried requests.

9. SHARED REDIS CONNECTION
   One ioredis client shared via REDIS_CLIENT token. Fewer TCP connections.

10. RATE LIMITING (Redis sliding window)
    10 req / 60s per IP. State in Redis — survives restarts.

11. VALIDATION PIPE (global)
    whitelist + forbidNonWhitelisted — strips / rejects unknown fields.

12. GRACEFUL SHUTDOWN
    app.enableShutdownHooks() — PrismaService disconnects cleanly on SIGTERM.

13. QR CODE AT CREATION TIME
    Generated once, stored as data URI in DB. No on-the-fly rendering.

14. TIME-SERIES ANALYTICS QUERY
    Separate Prisma query scoped to last 30 days with select: {createdAt}
    — fetches only one field instead of full ClickEvent rows.

================================================================================
DATA FLOW — END TO END
================================================================================

CREATING A SHORT URL
--------------------
  User → POST /url → UrlService.createShortUrl
    → validateExpiresAt → checkSpam (Safe Browsing) → bcrypt(password)
    → QRCode.toDataURL → setNx lock → prisma.url.create → release lock
    ← { shortCode, shortUrl, qrCode, hasPassword, maxClicks }

REDIRECTING A SHORT URL
-----------------------
  User clicks short URL
    → GET /:code → RateLimiter → RedirectService.getOriginalUrl
      → prisma.url.findUnique → expiry / password / maxClicks checks
      → Redis cache hit: return immediately
      → cache miss: set Redis with smart TTL
    → kafkaProducer.publishClickEvent (fire and forget)
       → KafkaConsumer: incr counter, UA parse, GeoIP, prisma.clickEvent.create, webhook
       → BatchProcessor (every 10s): flush Redis counters to DB
    ← HTTP 302

AUTHENTICATION FLOW
--------------------
  Register / Login:
    POST /auth/login → AuthService.login
      → checkBruteForce → prisma.findUnique → bcrypt.compare
      → generateTokens: sign JWT (jti, 15m), store refresh in Redis
      → controller: Set-Cookie refresh_token (httpOnly) + return access_token in body

  Refresh:
    POST /auth/refresh (browser sends httpOnly cookie automatically)
      → AuthService.refresh: validate refresh token, check revoked_family
      → delete old token, generateTokens with same familyId
      → Set-Cookie new refresh_token + return new access_token

  Logout:
    POST /auth/logout (Bearer header + httpOnly cookie)
      → revoke_family:{familyId}, blocklist:{jti}, clear cookie

  Protected request:
    GET /url/my-links with Authorization: Bearer <access_token>
      → JwtStrategy.validate: check blocklist:{jti} → return user

VIEWING ANALYTICS
-----------------
  User opens /:code/stats
    → GET /url/:code/stats → AnalyticsService.getStats
      → prisma.url.findUnique (include last 500 ClickEvents)
      → redis.get("clicks:{code}") → pendingClicks
      → aggregate browser/device/country counts
      → prisma.clickEvent.findMany (last 30 days, select createdAt)
      → group by day, fill zeros for days with no traffic
      → return full stats + top-10 lists + clicksPerDay[30]
  Frontend renders ClicksChart bar chart + breakdown bars

SECURITY CONSIDERATIONS
================================================================================

  1. CORS restricted to FRONTEND_URL env var (not wildcard *)
  2. URL validation — @IsUrl with require_protocol + require_tld
  3. Rate limiting — 10 req / 60s per IP via Redis-backed sliding window
  4. Input whitelist — ValidationPipe strips/rejects unknown fields
  5. Smart cache TTL — prevents stale expired URLs from being served
  6. Password protection — bcrypt(12) hashing; plaintext never stored
  7. Max-clicks enforcement — checked at redirect time (db + Redis buffer)
  8. Google Safe Browsing — malware/phishing detection at creation time
  9. rel="noopener noreferrer" on all external links
 10. API key hashing — SHA-256 stored; plaintext shown once; never recoverable
 11. Reserved path guard — health, metrics, url, auth, api never treated as codes
 12. HttpOnly Secure SameSite cookie — refresh token inaccessible to JavaScript/XSS
 13. Refresh token rotation — old token deleted on every refresh
 14. Token family revocation — replayed token triggers full session invalidation
 15. Login brute-force protection — 5 failures → 15-min lockout per email (Redis)
 16. Access token revocation — jti blocklist in Redis; immediate post-logout rejection
 17. bcrypt cost factor 12 — higher than default 10; resists offline cracking
